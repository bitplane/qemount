# D88 disk image

D88 is a container format for floppy-disk images of Japanese personal computers,
most associated with the NEC PC-88 and PC-98 series and also used for the Sharp
X1 and other micros. Unlike a flat sector dump (such as the Sharp
[2D](2d) image), D88 stores a header plus per-track, per-sector metadata, so it
can represent mixed densities, copy-protected layouts, variable sector sizes and
sectors in non-sequential order. A single file may also concatenate several disk
images.

## Structure

Each disk image starts with a header (688 bytes; 0x2B0 in the MAME loader)
followed by track data. Header fields (little-endian):

| Offset | Size | Field |
|--------|------|-------|
| 0x00 | 16 | Disk name / label |
| 0x1A | 1 | Write-protect flag (0x10 = protected) |
| 0x1B | 1 | Media type: 0x00 = 2D, 0x10 = 2DD, 0x20 = 2HD |
| 0x1C | 4 | Image size (header + data; mismatch signals a multi-disk file) |
| 0x20 | 4×164 | Track offset table (offset of each track from the start) |

Each track is a run of sectors; every sector is preceded by a 16-byte header
giving its cylinder, head, sector number, size code, sector count for the track,
density/deleted-data flags, and the sector's data length. The `.d77` extension
denotes a 2D image and `.1dd` a single-sided variant; `.d88` is the common name.
