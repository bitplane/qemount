# V7 Unix Filesystem

Created at Bell Labs for the PDP-11 in 1979 and released with V7 Unix, this is
the ancestor of most modern Unix filesystems.

## Characteristics:

- 14 character filename limit - hardcoded in the directory entry structure
- No symbolic links - those came with 4.2BSD in 1983
- 16-bit inodes - max ~65k files
- Block size: 512 bytes (fixed)
- Max filesystem size: ~1GB (24-bit block addresses * 512-byte blocks)
- Simple free list - not a bitmap, just a linked list of free blocks

## Structure

- Superblock at block 1 (offset 512), 440 bytes
- Inodes start at block 2 (offset 1024), 64 bytes each
- Root inode is inode 1, at offset 1088
- Directory entries: 16 bytes (2-byte inode + 14-byte name)
- Free blocks: linked list in s_free[50] array
- 24-bit block addresses (s_fsize <= 0x00ffffff)

### Superblock Layout (at offset 512)

| Offset | Size | Field      | Description                      |
|--------|------|------------|----------------------------------|
| 0      | 2    | s_isize    | First data zone                  |
| 2      | 4    | s_fsize    | Total zones in filesystem        |
| 6      | 2    | s_nfree    | Free blocks in cache (0-50)      |
| 8      | 200  | s_free[50] | Free block cache                 |
| 208    | 2    | s_ninode   | Free inodes in cache (0-100)     |
| 210    | 200  | s_inode    | Free inode cache                 |
| 428    | 6    | s_fname    | Filesystem name                  |
| 434    | 6    | s_fpack    | Pack name                        |

## Detection

V7 has no magic number. The Linux kernel (`fs/sysv/super.c`, `v7_sanity_check`)
detects it heuristically by checking structural plausibility:

- `s_nfree` (offset 518) as uint16 must be 0-50
- `s_ninode` (offset 720) as uint16 must be 0-100
- `s_fsize` (offset 514) as uint32 must be <= 0x00ffffff and > 0
- Root inode mode (offset 1088) masked with 0xF000 == 0x4000 (directory)
- Root inode size (offset 1092) > 0, multiple of 16, <= 16384

Byte ordering is PDP-11 middle-endian (3-4-1-2 for 32-bit values) on the
original platform, or little-endian on x86 ports. The kernel tries both.
