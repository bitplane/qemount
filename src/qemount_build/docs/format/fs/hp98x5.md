# HP 98x5 disk filesystem

This is the disk filesystem used by Hewlett-Packard's 9825, 9831 and 9845
desktop computers — programmable "calculator"-class machines of the late 1970s
and early 1980s. The three systems use closely related (but not identical) disk
directory formats, which MAME handles together. It is a structured,
record-based filesystem and is distinct from HP's later LIF (Logical Interchange
Format) used by the Series 80 / HP-IB world.

## Layout

The basic unit is a 256-byte *record*; the medium is treated as a linear
sequence of records starting at record 0. A flexible disk is organised in tracks
of 32 sectors (256 bytes each); a single-sided disk carries on the order of 65
data tracks (plus a couple of system tracks and a spare), a double-sided one
roughly 148.

Four areas make up the volume:

1. **System record** — the first record on the medium, holding disk metadata.
2. **File directory** — begins at sector 1 and occupies around 22 sectors.
3. **Availability table** — a small run of sectors (about 6) recording free
   space in fixed-size entries.
4. **File data area** — the remainder. When the directory and availability
   table are mirrored by a spare copy, the data area begins after the spare.

Directory entries are fixed 16-byte records: a 6-character (null-padded)
filename, the file's first sector, its sector count, and type/size attributes.
On the 9845 the directory list is terminated by an `0xFFFF` word. HP filenames
can carry a record-length and type suffix of the form `name[[.&bpr].ext]`, where
`bpr` is the bytes-per-record value.

## File types

The available file types differ by machine. The 9825 offers DATA, PROG, KEYS,
ALL and BPRG (often shown by their initials D, P, K, M, B); the 9845 adds BKUP,
BDAT, OPRM and ASMB among others. A `TEXT` view provides automatic ASCII↔DATA
translation for string-only files.
