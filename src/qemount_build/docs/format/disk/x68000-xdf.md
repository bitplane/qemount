# X68000 XDF disk image

This XDF is the raw floppy image format used by the **Sharp X68000** and its
Human68k operating system. It is unrelated to the similarly named IBM
[Extended Density Format](format/disk/ibmxdf) despite sharing the `.xdf`
extension and the "XDF" abbreviation — see the disambiguation note below.

The X68000 (and the closely related NEC PC-88VA) used a Japanese
2HD floppy layout of 77 cylinders, two heads, 8 sectors of 1024 bytes per track
at 360 RPM, which is exactly the geometry MAME's loader describes (MFM, 1.2 MB
class). The total comes to about 1,232 KB of sector data. An XDF file is simply
that geometry written out as a flat, header-less sector dump in track/head/
sector order, with no signature — the same bytes a real Human68k disk holds,
which is why it is also seen carrying the `.hdm` and `.2hd` extensions common to
Japanese 1.2 MB floppy images. Tools such as `dis68k` and `fathuman` list and
unpack the Human68k FAT filesystem stored inside.

In MAME the loader is implemented on the µPD765 sector-dump framework and the
`FLOPPY_XDF_FORMAT` instance is registered by the Sharp X68000 (`x68k`) and NEC
PC-88VA drivers. The MAME source notes that the gap values are unverified.

## Disambiguation: this is not IBM XDF

Two different formats are abbreviated "XDF":

- **This one** — the Sharp X68000 / Human68k 1.2 MB raw floppy image
  (77×2×8×1024), a plain Japanese 2HD sector dump. Implemented in MAME's
  `xdf_dsk.cpp`.
- **IBM's eXtended Density Format**, documented at
  [`format/disk/ibmxdf`](format/disk/ibmxdf) — a PC DOS 7 / OS/2 high-capacity
  scheme that packs mixed-size sectors onto an ordinary 1.44 MB PC floppy.
  Implemented in MAME's separate `ibmxdf_dsk.cpp`.

They share only the three letters; the geometry, encoding, host platform, and
filesystem are all different.
