# PC-98 raw floppy image

A raw, headerless sector dump of a floppy disk from the NEC PC-9800 (PC-98)
series, the architecture that dominated Japanese personal computing through the
1980s and 1990s. The file is simply every sector of the disk laid out in
cylinder/head/sector order, with no header, magic, or metadata in front of it —
the same idea as a plain `.img`, but using PC-98 disk geometries. The `.hdm`
extension is the one most specific to this family (a PC-98 2HD floppy dump); the
other extensions are shared with generic PC images.

Because there is no header, MAME identifies the geometry from the file's size,
matching it against a table of known PC-98 configurations: 5.25-inch disks from
160 KB up to 1.2 MB and 3.5-inch disks from 720 KB up to 2.88 MB, with sector
sizes of 256, 512 or 1024 bytes across 40 or 80 cylinders, single- or
double-sided, MFM-encoded. The native PC-98 2HD layout (commonly 256 bytes ×
26 sectors, or 1024 bytes × 8 sectors, over 77 cylinders × 2 heads, ~1.23 MB) is
the headline case; the loader builds tracks through MAME's `upd765` formatter,
matching the uPD765-family controller these machines used.

This sits at the plain-dump end of the PC-98 image spectrum. Richer PC-98
container formats carry per-sector metadata and can represent mixed densities or
copy protection: [D88](d88), the Virtual98 [FDD](fdd) format, [DIP](dip), and the
NFD format (not yet catalogued). The Anex86 [FDI](pc98fdi) format is essentially
this same raw sector body with a 4096-byte header bolted on the front.

No detection rule is asserted: the format is headerless and identified only by
exact file size, which collides badly across the catalogue.

## Distinguishing from generic PC images

This format shares the `.dsk`/`.ima`/`.img`/`.ufi`/`.360` extensions with the
[generic IBM PC raw image](pc-img). The two are distinct geometry families —
PC-98 disks use 256- and 1024-byte sectors and the 77-cylinder 2HD layout,
whereas the PC family is built around 512-byte sectors — but neither has a
header, so an image's true identity comes from its geometry, not its extension.
