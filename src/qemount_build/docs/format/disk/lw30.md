# Brother LW-30 floppy image

A decoded sector image of the 3.5" floppies used by the Brother LW-30, an
electronic typewriter / word-processing appliance (a small Z180-class machine
with an LCD and a built-in daisywheel printer). The LW-30 does **not** use
ordinary 720 KB/1.44 MB PC floppies: it writes a Brother-specific
group-coded-recording (GCR) scheme in software that standard PC floppy
controllers cannot read directly.

## Structure

The `.img` file is a flat sector dump; the GCR encoding lives on the physical
medium, and MAME's loader generates/parses that flux when converting to and from
the image. Per MAME's geometry:

- 78 tracks, single-sided
- 12 sectors per track
- 256 bytes per sector (≈ 234 KB of sector data)

On the medium, each track carries a `0xaa` lead-in, then twelve sectors; each
sector has a GCR sync mark, a 5-bits-to-8-bits encoded 256-byte payload with a
checksum, and trailing padding. Sector ordering is interleaved by track position.
There is no file-level magic number in the decoded `.img`.

Independent descriptions of the Brother WP/LW GCR family agree on twelve 256-byte
sectors per track and a ~240 KB double-density capacity; some sources describe
the higher-capacity members of the family as double-sided, whereas MAME models
the LW-30 image as 78 single-sided tracks. Trust the per-model details when
handling a specific disk.
