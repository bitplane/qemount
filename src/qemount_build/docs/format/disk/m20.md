# Olivetti M20 floppy image

A flat sector image of the 5.25" floppies used by the Olivetti M20, an Italian
business microcomputer built around the Zilog Z8000 16-bit CPU and launched in
1982 (it ran Olivetti's PCOS operating system). The image is a raw, headerless
sector dump.

## Structure

Per MAME's loader the disk geometry is:

- 35 tracks, 2 heads
- 16 sectors per track, 256 bytes per sector
- Total image size 286,720 bytes (35 × 2 × 16 × 256)

The M20 format is mixed-density. Track 0 / head 0 is written in **FM** with
128-byte sectors (the rest of the disk is **MFM** with 256-byte sectors); in the
image, those first-track sectors are stored in 256-byte slots with only the first
128 bytes meaningful. This FM boot track is one reason ordinary PC floppy
controllers struggle with M20 media.

There is no magic number; MAME recognises the format by its fixed size and
geometry. (Note that an online write-up reasoning from the 286,720-byte size
assumed 512-byte sectors — that arithmetic does not hold; the format uses
256-byte sectors, which is what both MAME and Olivetti documentation describe.)
