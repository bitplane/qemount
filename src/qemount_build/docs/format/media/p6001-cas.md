# PC-6001 cassette image

A cassette-tape image for the NEC PC-6001, the entry-level 8-bit home computer
NEC launched in Japan in 1981 (the start of the PC-6000 series). The `.cas` file
holds the logical byte stream of a program saved from cassette — it is the data
that travelled over the tape, stripped of the analogue waveform, much like a
sector dump of a floppy. The same payload is also seen under the `.p6`
extension, which some emulators and tools require.

This is a tape image with no filesystem to mount — it is catalogued here for
identification and cross-reference (no driver).

The PC-6001 cassette world has several related image types: the bare-bytes
`.cas`/`.p6` form documented here, a richer `.p6t` form that tries to preserve
the raw audio (and so copy-protected or speed-modified tapes), and a partly
ASCII BASIC dump. Only the bare-bytes form is what MAME's loader handles.

## Structure

MAME treats the `.cas` file as the program's raw data bytes and synthesises a
tape waveform from them: each byte expands to eight audio samples at an 8000 Hz
sample rate, with each bit driving the output to a fixed high or low amplitude.
The file itself carries no header or magic — it is just the sequence of data
bytes. Independent PC-6001 documentation describes the on-tape framing as UART
style (one start bit, eight data bits sent least-significant-bit first, two stop
bits), but that framing is regenerated on playback and is not stored in the
`.cas` file.

Because the file is a headerless byte stream with no internal navigable
structure, no detection rule is asserted.
