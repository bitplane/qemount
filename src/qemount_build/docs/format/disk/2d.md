# 2D (Sharp floppy image)

A raw, headerless sector image of a "2D"-type 5.25" floppy as used by early
Sharp machines (notably the Sharp X1). "2D" is Japanese floppy nomenclature for
a double-sided, double-density disk, and the image is simply a straight dump of
that disk's sectors with no container header or metadata.

## Geometry

Fixed geometry, MFM encoding:

| Property | Value |
|----------|-------|
| Tracks (cylinders) | 40 |
| Sides | 2 |
| Sectors / track | 16 |
| Bytes / sector | 256 |
| Sector numbering | starts at 1 |
| Total size | 327,680 bytes (320 KB) |

Because the format carries no header, there are no magic bytes to identify it;
in practice it is recognised by its fixed size and geometry, the `.2d`
extension, and context. Sharp X1 collections also commonly use the richer
[D88](d88) container, which does store geometry per track.
