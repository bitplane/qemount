# G64 GCR disk image (.g64/.g71)

`.g64` is the bitstream-level preservation format for Commodore 1541/1571
floppies. Where the [`.d64` family](commodore-disk.md) stores decoded 256-byte
DOS blocks, a G64 holds each track as the raw GCR (Group Code Recording)
bitstream the drive's read head actually sees — sync marks, header and data GCR
groups, gaps and all. It is the low-level GCR sibling of the sector-level `.dXX`
images: G64 sits one layer below D64, and CBM DOS (see [`fs/cbmdos`](../fs/cbmdos.md))
is the filesystem that lives inside the sectors a G64 encodes.

The format exists because a small set of C64 titles use copy protection or custom
fast-loaders that depend on non-standard track layouts — extra sectors, odd sync
patterns, half-tracks, or deliberately weak bits — that a decoded D64 simply
cannot represent. Capturing the GCR bitstream preserves those tricks. It was
introduced in the late 1990s for the VICE emulator and has been the de-facto GCR
container for C64 disk preservation since. The `.g71` variant carries both sides
of a 1571 disk; `.g41` is the matching single-density 1541 spelling MAME accepts.

## Structure

The file begins with a fixed header:

- An 8-byte signature, the ASCII string `GCR-1541`.
- A 1-byte version number (currently `0`; MAME rejects anything else).
- A 1-byte count of tracks/half-tracks stored (typically 84 — 42 tracks plus
  the 42 in-between half-tracks the 1541 stepper can reach).
- A 2-byte little-endian maximum track size in bytes (commonly 7928).

Two parallel tables follow, each with one 4-byte little-endian entry per
(half-)track:

- a **track offset table** — the absolute file offset of each track's GCR data,
  or `0` if that track is not present; and
- a **speed-zone table** — either a value 0–3 selecting one of the drive's four
  speed zones for the whole track, or an offset to a per-byte speed map.

Each track block is itself prefixed by a 2-byte little-endian length and then the
packed GCR bitstream (8 bits per byte). The four speed zones correspond to the
1541's zone-bit recording, where outer tracks are clocked faster than inner ones;
MAME's cell sizes for zones 0–3 are 4000, 3750, 3500 and 3250 (derived from the
16 MHz controller clock).

## Detection

A G64 always begins with the 8-byte ASCII signature `GCR-1541` at offset 0,
followed by a version byte of `0`. MAME's loader and the published VICE/G64 format
specifications agree on this signature and version, so it is a reliable
identifier. (The double-sided `.g71` images use the same `GCR-1541` magic.)
