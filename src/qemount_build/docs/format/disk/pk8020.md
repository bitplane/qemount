The `.kdi` (Korvet Disk Image) file is a floppy image for the PK-8020 "Korvet",
a Soviet Z80-class educational microcomputer deployed in classrooms in the late
1980s. The machine could run CP/M, and KDI is the standard sector-dump container
for its diskettes.

## Structure

MAME models the medium as a 5.25" double-sided, quad-density MFM floppy using a
WD177x-style layout: 80 tracks, 2 heads, 5 sectors per track of 1024 bytes,
giving a 5 × 1024 × 80 × 2 = 819,200-byte image at a 2000 kbps data rate. (The
loader carries a TODO noting its gap sizes — 60, 22, 116 — still need
verification.) Beyond the raw geometry, the Korvet community documents that the
image's first (zeroth) sector holds a format-description block used for
identification: the leading bytes carry bootloader information, a following group
describes the physical disk device and a further group describes the logical
disk, with a CRC included so a reader can recognise the format. The body is an
ordinary CP/M filesystem.

(The fixed geometry is taken from the MAME loader; the sector-0 descriptor layout
is taken from Korvet community documentation. The two sources agree on the
overall image but describe no single fixed file-magic signature, so no formal
detection rule is asserted here.)
