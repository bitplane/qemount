# 86F (86Box floppy surface image)

The native floppy *surface* image format of the 86Box PC emulator. Instead of
storing decoded sectors, 86F records the FM/MFM flux transitions of the disk
surface, optionally with per-cell surface description data. This lets it
represent weak bits and other low-level features used by copy protection that a
plain sector image (like a raw `.img`) cannot.

## Characteristics

- Surface-level FM/MFM transition data, not decoded sectors
- Optional surface description data for weak/fuzzy bits
- Hole density DD / HD / ED (including ED at 2000 kbps)
- One or two sides; write-protect flag
- Optional RPM slowdown (0 / 1 / 1.5 / 2%) for protection schemes

## Detection

The header begins with a 4-byte signature `86BF` at offset 0, followed by a
minor version byte (`0x0C`) at offset 4 and a major version byte (`0x02`) at
offset 5, then a 16-bit disk-flags field at offset 6. (Signature confirmed by
both the MAME loader and the 86Box format specification.)
